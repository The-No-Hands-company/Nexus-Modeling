# v0.5 Planning — DLSS Ray Reconstruction + RT Shader Pipeline

Status: **Planning**. Targets the first release with a fully wired scene-level RT path:
BVH integration, ray-generation shaders, and DLSS Ray Reconstruction.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.4-planning.md](v0.4-planning.md).

---

## Motivation

v0.4 delivered `createRayTracingPipeline`, live Vulkan RT dispatch tests, and neural
renderer backend reporting. The remaining gap to a production RT frame is:

1. Scene-level BVH: connecting `SceneGraph` geometry to BLAS/TLAS resources.
2. Ray-generation shader integration: binding the TLAS in a descriptor set and
   dispatching `traceRays` with real scene geometry.
3. DLSS Ray Reconstruction (DLSS-RR): NVIDIA's denoising path tuned for RT content.
4. Mesh shader production path: finalize `createMeshShaderPipeline` integration
   with `Renderer::render()` scheduler path.

---

## Scope

### Track 1 — Scene BVH Integration

| Deliverable | Notes |
|---|---|
| `SceneGraph::buildAccelStructs(IDevice&)` | Walks visible nodes, builds a BLAS per mesh, builds a TLAS over all BLASes. Returns `AccelStructHandle` for the TLAS. |
| `Renderer::setSceneTLAS(AccelStructHandle)` | Stores a non-owning TLAS handle; bound as descriptor when RT is enabled. |
| `FrameStats::tlasInstanceCount` | Number of BLAS instances in the current frame's TLAS. |
| BLAS caching | Per-node BLAS rebuilt only when mesh content changes (dirty flag on SceneNode). |
| Null-backend tests | `test_SceneBVH.cpp` — 8 tests: build, rebuild, node add/remove, empty scene, TLAS handle round-trip on Renderer. |

### Track 2 — Ray-Generation Shader Integration

| Deliverable | Notes |
|---|---|
| RT descriptor layout | TLAS bound at set=0 binding=0; output image at set=0 binding=1. |
| `Renderer::render()` RT path | When `rtReflectionsActive` conditions all hold: bind TLAS descriptor + RT pipeline, dispatch `traceRays`. |
| `test_VulkanRTDispatch.cpp` additions | `TraceRaysWithSceneTLASOnTier1`: builds a TLAS from the scene, runs RT dispatch, asserts `rtReflectionsActive + tlasInstanceCount > 0`. |

### Track 3 — DLSS Ray Reconstruction

| Deliverable | Notes |
|---|---|
| `DenoiserBackend::DLSS_RR` enum value | New variant distinct from `DLSS4` denoiser. |
| `DLSSPlugin` Ray Reconstruction path | Calls `NVSDK_NGX_DLSS_EvaluateFeature` with `NVSDK_NGX_Feature_RayReconstruction`. |
| `NeuralBackend::DLSS_RR` factory entry | `NeuralRendererFactory::create(DLSS_RR, device)`. |
| CI skip contract | Skips when `NEXUS_ENABLE_DLSS=OFF` or SDK absent. |

### Track 4 — Mesh Shader Production Path Completion

| Deliverable | Notes |
|---|---|
| `Renderer::setMeshShaderPipeline(PipelineHandle)` / `meshShaderPipeline()` | Non-owning slot mirroring `rayTracingPipeline`. |
| `RendererSettings::enableMeshShaders` flag | Gates `drawMeshTasks` dispatch. |
| `FrameStats::meshShaderDrawCalls` | Counter for mesh-shader dispatches. |
| `test_MeshShaderProductionPath.cpp` | 8 tests: default state, pipeline round-trip, mode gate, Null backend no-crash. |

## Prerequisites

| Requirement | Notes |
|---|---|
| Vulkan device with `VK_KHR_ray_tracing_pipeline` + `VK_KHR_acceleration_structure` | Already required from v0.4 |
| DLSS 4.x SDK with Ray Reconstruction support | NGX feature flag `NVSDK_NGX_Feature_RayReconstruction` |
| Mesh shader support (`VK_EXT_mesh_shader` or `VK_NV_mesh_shader`) | `caps().meshShadersSupported` |

---

## Out of Scope for v0.5

- Full path tracer (material BRDFs, multiple-bounce lighting, importance sampling).
- AMD FSR 3 frame generation.
- macOS MoltenVK RT path.
- Multi-GPU rendering.

---

## Exit Criteria

1. `SceneGraph::buildAccelStructs` produces a valid TLAS on both Null and Vulkan backends.
2. `TraceRaysWithSceneTLASOnTier1` passes on a Tier-1 RT device.
3. `NeuralRendererFactory::create(DLSS_RR, device)` returns non-null; skips cleanly when SDK absent.
4. `RendererSettings::enableMeshShaders` gates mesh-shader dispatch correctly.
5. All 8 `test_SceneBVH` and 8 `test_MeshShaderProductionPath` tests pass on Null backend.
6. Null-backend full suite still at 100% (no regressions).
