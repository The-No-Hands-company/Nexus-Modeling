# v0.19 Planning — Polarisation Rendering, Fluorescence, Spectral Upsampling

Status: **Planning**. Three passes completing the advanced spectral transport stack
deferred from v0.18.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.18-planning.md](v0.18-planning.md).

---

## Motivation

v0.18 delivered multi-spectral N-wavelength rendering, bidirectional path tracing, and
histogram-based auto white balance. Three items were explicitly deferred:

1. **Polarisation Rendering** — tracks Stokes vector state through reflection, refraction,
   and scattering events; enables accurate rendering of polarised glass, LCD panels,
   metallic surfaces, and Brewster-angle effects.
2. **Fluorescence / Phosphorescence Simulation** — models wavelength-shifting re-emission
   from fluorescent materials (instantaneous) and phosphorescent materials (time-delayed);
   requires spectral-to-spectral re-emission matrix sampled during multi-spectral passes.
3. **Spectral Upsampling from RGB Assets** — converts legacy RGB albedo/emissive textures
   to plausible spectral distributions using Smits or sigmoid upsampling; enables spectral
   rendering of existing asset libraries without artist re-authoring.

## Out of Scope for v0.19

## Exit Criteria

1. `enablePolarisation` triggers Stokes vector dispatch; `polarisationActive` and
   `polarisationRayCount` populated.
2. `enableFluorescence` triggers re-emission pass; `fluorescenceActive` and
   `fluorescenceEmissionBands` populated.
3. `enableSpectralUpsampling` triggers RGB→spectral upsampling; `spectralUpsamplingActive`
   and `spectralUpsamplingMethod` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
