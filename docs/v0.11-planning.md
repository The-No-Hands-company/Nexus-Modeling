# v0.11 Planning — Subsurface Scattering, Contact Shadows, Tiled Deferred Lighting

Status: **Planning**. Three rendering passes that extend the lighting model beyond
the per-pixel ambient/composite pipeline established in v0.8–v0.10.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.10-planning.md](v0.10-planning.md).

---

## Motivation

v0.10 completed IBL, OIT, and FSR 4. Three lighting-quality passes remain before the
kernel reaches production-grade fidelity for organic and complex scenes:

1. **Subsurface Scattering (SSS)** — screen-space separable SSS blur applied to
   GBuffer materials flagged as translucent; controlled by scatter radius and profiles.
2. **Screen-Space Contact Shadows** — short-range ray march in the depth buffer
   immediately around shadow-casting geometry; fills the gap between shadow map
   cascades and contact geometry.
3. **Tiled Deferred Lighting** — 16×16-pixel light tile classification pass that
   feeds the composite with a per-tile light list; required for scenes with > 64
   dynamic lights without draw-call explosion.

## Scope

### Track 1 — Subsurface Scattering (SSS)

| Deliverable | Notes |
|---|---|
| `SSSSettings` struct | `enabled`, `scatterRadius`, `profileCount`, `blurPasses` |
| `Renderer::setSSSSettings` / `sssSettings()` | Stored on renderer |
| `FrameStats::sssActive` | `true` when SSS blur dispatch ran |
| `FrameStats::sssBlurPasses` | Blur passes dispatched this frame |
| Compute dispatch in `render()` | After GBuffer; 8×8 tiles; gated on `enableSSS` |
| Null-backend tests | `test_SubsurfaceScattering.cpp` — 8 tests |

### Track 2 — Screen-Space Contact Shadows

| Deliverable | Notes |
|---|---|
| `ContactShadowSettings` struct | `enabled`, `rayLength`, `sampleCount`, `thickness` |
| `Renderer::setContactShadowSettings` / `contactShadowSettings()` | Stored on renderer |
| `FrameStats::contactShadowsActive` | `true` when contact shadow dispatch ran |
| `FrameStats::contactShadowRayCount` | `width × height` rays cast this frame |
| Compute dispatch in `render()` | After shadow pass; before composite |
| Null-backend tests | `test_ContactShadows.cpp` — 8 tests |

### Track 3 — Tiled Deferred Lighting

| Deliverable | Notes |
|---|---|
| `TiledLightingSettings` struct | `enabled`, `tileSize`, `maxLightsPerTile` |
| `Renderer::setTiledLightingSettings` / `tiledLightingSettings()` | Stored on renderer |
| `FrameStats::tiledLightingActive` | `true` when tile classification ran |
| `FrameStats::lightTileCount` | Total 16×16 tiles classified this frame |
| `FrameStats::maxLightsPerTile` | Peak lights in any tile this frame |
| Compute dispatch in `render()` | After GBuffer; tile classification before composite |
| Null-backend tests | `test_TiledDeferredLighting.cpp` — 8 tests |

## Out of Scope for v0.11

- Full pre-integrated SSS with LUT texture.
- Ray-traced contact shadows.
- GPU-driven clustered lighting (3-D clusters deferred to v0.12).

---

## Exit Criteria

1. `enableSSS` triggers SSS blur; `sssActive` and `sssBlurPasses` populated.
2. `enableContactShadows` triggers contact shadow march; `contactShadowsActive`
   and `contactShadowRayCount` populated.
3. `enableTiledLighting` triggers tile classification; `tiledLightingActive`,
   `lightTileCount`, and `maxLightsPerTile` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
