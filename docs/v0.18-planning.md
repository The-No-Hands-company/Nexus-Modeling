# v0.18 Planning — Multi-Spectral Rendering, BDPT, Auto White Balance

Status: **Planning**. Three passes completing the spectral and photon transport stack
deferred from v0.17, plus histogram-based perceptual white balance.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.17-planning.md](v0.17-planning.md).

---

## Motivation

v0.17 delivered hero wavelength spectral dispersion, progressive photon mapping, and
auto-exposure. Three items were explicitly deferred:

1. **Multi-Spectral Rendering** — full N-wavelength path tracing beyond the single hero
   wavelength; enables accurate color dispersion, fluorescence, and iridescence effects.
   Deferred from v0.17.
2. **Bidirectional Photon Mapping (BDPT)** — combines forward path tracing with photon
   map gather using bidirectional connections; superior convergence for caustics and
   enclosed scenes. Deferred from v0.17.
3. **Histogram-Based Auto White Balance** — GPU luminance/chrominance histogram over the
   rendered frame followed by a gray-world or white-patch correction; feeds a 3×3 color
   matrix into the tone mapping pass each frame.

## Scope

### Track 1 — Multi-Spectral Rendering

| Deliverable | Notes |
|---|---|
| `MultiSpectralSettings` struct | `enabled`, `wavelengthBands`, `minWavelengthNm`, `maxWavelengthNm`, `samplesPerBand` |
| `Renderer::setMultiSpectralSettings` / `multiSpectralSettings()` | Stored on renderer |
| `RendererSettings::enableMultiSpectral` | Gate flag (default `false`) |
| `FrameStats::multiSpectralActive` | `true` when multi-spectral dispatch ran |
| `FrameStats::multiSpectralBandCount` | Wavelength bands evaluated this frame |
| Compute dispatch in `render()` | After spectral dispersion pass; one pass per band × samplesPerBand |
| Null-backend tests | `test_MultiSpectral.cpp` — 8 tests |

### Track 2 — Bidirectional Photon Mapping (BDPT)

| Deliverable | Notes |
|---|---|
| `BDPTSettings` struct | `enabled`, `lightPathLength`, `eyePathLength`, `mis` (MIS weighting) |
| `Renderer::setBDPTSettings` / `bdptSettings()` | Stored on renderer |
| `RendererSettings::enableBDPT` | Gate flag (default `false`) |
| `FrameStats::bdptActive` | `true` when BDPT dispatch ran |
| `FrameStats::bdptConnectionCount` | Light–eye path connections evaluated this frame |
| Compute dispatch in `render()` | Light path trace pass + eye path trace pass + connection pass |
| Null-backend tests | `test_BDPT.cpp` — 8 tests |

### Track 3 — Histogram-Based Auto White Balance

| Deliverable | Notes |
|---|---|
| `AutoWhiteBalanceSettings` struct | `enabled`, `method` (GrayWorld/WhitePatch), `strength`, `adaptationSpeed` |
| `AutoWhiteBalanceMethod` enum | `GrayWorld`, `WhitePatch` |
| `Renderer::setAutoWhiteBalanceSettings` / `autoWhiteBalanceSettings()` | Stored on renderer |
| `RendererSettings::enableAutoWhiteBalance` | Gate flag (default `false`) |
| `FrameStats::autoWhiteBalanceActive` | `true` when histogram + correction ran |
| `FrameStats::autoWhiteBalanceMethod` | Active correction method this frame |
| Compute dispatch in `render()` | After auto-exposure; histogram reduce then matrix update |
| Null-backend tests | `test_AutoWhiteBalance.cpp` — 8 tests |

## Out of Scope for v0.18

## Exit Criteria

1. `enableMultiSpectral` triggers N-band dispatch; `multiSpectralActive` and
   `multiSpectralBandCount` populated.
2. `enableBDPT` triggers light path + eye path + connection passes; `bdptActive`
   and `bdptConnectionCount` populated.
3. `enableAutoWhiteBalance` triggers histogram + correction; `autoWhiteBalanceActive`
   and `autoWhiteBalanceMethod` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
