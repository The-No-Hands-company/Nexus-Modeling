# v0.13 Planning — Ray-Traced GI, Atmospheric Scattering, Light Shafts

Status: **Planning**. Three rendering passes that extend the kernel into
physically-based atmospheric and indirect lighting beyond screen-space techniques.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.12-planning.md](v0.12-planning.md).

---

## Motivation

v0.12 completed clustered lighting, SSGI, and decals. Three passes remain before
the kernel can handle large outdoor and architectural scenes at production fidelity:

1. **Ray-Traced Global Illumination (RTGI)** — hardware RT one-bounce diffuse and
   specular GI; feeds a screen-space denoised irradiance cache. Explicitly deferred
   from v0.12.
2. **Atmospheric Scattering** — Rayleigh + Mie transmittance LUT evaluated
   per-frame; drives sky colour and aerial perspective for geometry beyond the
   volumetric froxel range.
3. **Light Shafts (God Rays)** — radial blur pass in post from an occluder mask
   toward the sun screen-position; composited additively before tone mapping.

## Scope

### Track 1 — Ray-Traced Global Illumination (RTGI)

| Deliverable | Notes |
|---|---|
| `RTGISettings` struct | `enabled`, `raysPerPixel`, `maxBounces`, `denoised` |
| `Renderer::setRTGISettings` / `rtgiSettings()` | Stored on renderer |
| `FrameStats::rtgiActive` | `true` when RTGI dispatch ran |
| `FrameStats::rtgiRaysDispatched` | `width × height × raysPerPixel` |
| Compute dispatch in `render()` | After GBuffer; gated on `enableRTGI` |
| Null-backend tests | `test_RayTracedGI.cpp` — 8 tests |

### Track 2 — Atmospheric Scattering

| Deliverable | Notes |
|---|---|
| `AtmosphericScatteringSettings` struct | `enabled`, `rayleighScale`, `mieScale`, `sunZenithAngle` |
| `Renderer::setAtmosphericScatteringSettings` / `atmosphericScatteringSettings()` | Stored on renderer |
| `FrameStats::atmosphericScatteringActive` | `true` when transmittance LUT dispatch ran |
| `FrameStats::atmosphericLUTSize` | Side length of transmittance LUT dispatched |
| Compute dispatch in `render()` | Before composite; LUT rebuild when dirty |
| Null-backend tests | `test_AtmosphericScattering.cpp` — 8 tests |

### Track 3 — Light Shafts (God Rays)

| Deliverable | Notes |
|---|---|
| `LightShaftSettings` struct | `enabled`, `sampleCount`, `decay`, `exposure` |
| `Renderer::setLightShaftSettings` / `lightShaftSettings()` | Stored on renderer |
| `FrameStats::lightShaftsActive` | `true` when radial blur pass ran |
| `FrameStats::lightShaftSampleCount` | Radial samples dispatched this frame |
| Compute dispatch in `render()` | After composite; before tone mapping |
| Null-backend tests | `test_LightShafts.cpp` — 8 tests |

## Out of Scope for v0.13

- Multi-bounce RTGI path (deferred to v0.14).
- Precomputed atmospheric LUT atlas across time-of-day range.
- Volumetric light shafts with full froxel integration.

---

## Exit Criteria

1. `enableRTGI` triggers RT GI dispatch; `rtgiActive` and `rtgiRaysDispatched` populated.
2. `enableAtmosphericScattering` triggers LUT dispatch; `atmosphericScatteringActive`
   and `atmosphericLUTSize` populated.
3. `enableLightShafts` triggers radial blur pass; `lightShaftsActive` and
   `lightShaftSampleCount` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
