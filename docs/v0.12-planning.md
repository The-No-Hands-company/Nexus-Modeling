# v0.12 Planning — Clustered Lighting, SSGI, Decal Rendering

Status: **Planning**. Three rendering passes that complete the lighting and surface detail
model established in v0.8–v0.11.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.11-planning.md](v0.11-planning.md).

---

## Motivation

v0.11 delivered SSS, contact shadows, and tiled deferred lighting. Three passes remain
before the kernel reaches production-grade fidelity for dense and complex scenes:

1. **GPU-Driven Clustered Lighting** — 3-D light cluster classification in view-space
   froxels (X×Y×Z); replaces tiled deferred for scenes with > 256 lights or multi-layer
   geometry. Explicitly deferred from v0.11.
2. **Screen-Space Global Illumination (SSGI)** — one-bounce diffuse GI reconstructed
   from the depth buffer and GBuffer normals; fills in inter-object bounce lighting
   without full path tracing.
3. **Decal Rendering Pass** — projected decal resolve after GBuffer; writes into
   albedo/normal/roughness channels before lighting composite; supports up to
   `maxDecals` per frame.

## Scope

### Track 1 — GPU-Driven Clustered Lighting

| Deliverable | Notes |
|---|---|
| `ClusteredLightingSettings` struct | `enabled`, `clusterDimX`, `clusterDimY`, `clusterDimZ`, `maxLightsPerCluster` |
| `Renderer::setClusteredLightingSettings` / `clusteredLightingSettings()` | Stored on renderer |
| `FrameStats::clusteredLightingActive` | `true` when cluster classification dispatch ran |
| `FrameStats::lightClusterCount` | Total X×Y×Z clusters classified this frame |
| `FrameStats::clusteredMaxLightsPerCluster` | Peak lights in any cluster this frame |
| Compute dispatch in `render()` | After GBuffer; cluster assignment before composite |
| Null-backend tests | `test_ClusteredLighting.cpp` — 8 tests |

### Track 2 — Screen-Space Global Illumination (SSGI)

| Deliverable | Notes |
|---|---|
| `SSGISettings` struct | `enabled`, `rayCount`, `rayLength`, `maxRoughness` |
| `Renderer::setSSGISettings` / `ssgiSettings()` | Stored on renderer |
| `FrameStats::ssgiActive` | `true` when SSGI gather dispatch ran |
| `FrameStats::ssgiRayCount` | `width × height × rayCount` rays dispatched this frame |
| Compute dispatch in `render()` | After GBuffer; before composite |
| Null-backend tests | `test_SSGI.cpp` — 8 tests |

### Track 3 — Decal Rendering Pass

| Deliverable | Notes |
|---|---|
| `DecalSettings` struct | `enabled`, `maxDecals`, `atlasResolution` |
| `Renderer::setDecalSettings` / `decalSettings()` | Stored on renderer |
| `FrameStats::decalsActive` | `true` when decal projection pass ran |
| `FrameStats::decalCount` | Decals projected this frame (capped at `maxDecals`) |
| Compute dispatch in `render()` | After GBuffer; writes into GBuffer channels |
| Null-backend tests | `test_DecalRendering.cpp` — 8 tests |

---

## Out of Scope for v0.12

- Hardware ray-traced GI (deferred to v0.13).
- Decal normal blending modes beyond simple overwrite.
- Dynamic cluster resolution based on light density.

---

## Exit Criteria

1. `enableClusteredLighting` triggers cluster classification; `clusteredLightingActive`,
   `lightClusterCount`, and `clusteredMaxLightsPerCluster` populated.
2. `enableSSGI` triggers SSGI gather; `ssgiActive` and `ssgiRayCount` populated.
3. `enableDecals` triggers decal projection; `decalsActive` and `decalCount` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
