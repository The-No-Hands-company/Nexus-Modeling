# v0.6 Planning — FSR 3, NGX Wiring, RT Shader Binding Tables

Status: **Planning**. Picks up the remaining `(void)preferFSR` stub from the neural
renderer, completes NGX evaluation parameter wiring, and adds RT shader binding table
(SBT) support for multi-material ray tracing.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.5-planning.md](v0.5-planning.md).

---

## Motivation

v0.5 delivered full scene BVH integration, DLSS Ray Reconstruction, RT descriptor
binding, and mesh shader production path. The remaining gaps before a production RT
frame are:

1. **FSR 3 upscaler** — `UpscalerBackend::FSR3` and `DenoiserBackend::FSR3` are in the
   enums but `createNeuralRenderer` ignores `preferFSR` entirely. AMD FidelityFX SR 3
   fills the upscaler slot for non-NVIDIA hardware.
2. **NGX parameter wiring** — `DLSSPlugin::denoise()` and `upscale()` both pass through
   without invoking the NGX evaluation API. Full wiring closes the gap between "SDK
   present" and "SDK doing work."
3. **RT shader binding tables** — `traceRays` dispatches with a single ray-gen shader.
   Multi-material scenes need hit/miss shader groups registered in a shader binding table
   (SBT) before `traceRays` is useful end-to-end.

## Scope

### Track 1 — FSR 3 Upscaler Integration

| Deliverable | Notes |
|---|---|
| `FSR3Plugin` class | Wraps the FidelityFX SDK context; `available()` gated on `NEXUS_ENABLE_FSR3` |
| `NeuralBackend::FSR3` factory case | `NeuralRendererFactory::create(FSR3, device)` → `FSR3Plugin` or Bilinear fallback |
| `UpscalerBackend::FSR3` reporting | `FSR3Plugin::activeUpscaler()` returns `FSR3` when SDK initialises |
| `DenoiserBackend::FSR3` reporting | `FSR3Plugin::activeDenoiser()` returns `FSR3` when SDK initialises |
| `perf_smoke` token | `--neural-backend fsr3` accepted |
| Null-backend tests | `test_FSR3Upscaler.cpp` — 7 tests |

### Track 2 — NGX Evaluation Parameter Wiring

| Deliverable | Notes |
|---|---|
| `DLSSPlugin::denoise()` NGX wiring | Full `NVSDK_NGX_VULKAN_EvaluateFeature` call with `InColor`, `InDepth`, `InMotionVectors`; falls back to passthrough when feature handle is null |
| `DLSSPlugin::upscale()` NGX wiring | Same API path for the DLSS upscaler feature |
| `NGX_Result` error propagation | Logs NGX result code; marks `m_ngxAvailable = false` on repeated failure |
| Vulkan integration test extension | `DLSSEvaluateDoesNotCrashOnTier1` — verifies the wired path survives a single call with stub texture handles on real hardware |

### Track 3 — RT Shader Binding Tables

| Deliverable | Notes |
|---|---|
| `ShaderBindingTableDesc` struct | `rayGenShader`, `missShaders[]`, `hitGroups[]` (each with closest-hit + optional any-hit) |
| `IDevice::createShaderBindingTable(ShaderBindingTableDesc)` | Returns `SBTHandle`; Null backend returns a valid stub |
| `SBTHandle` type | `Handle<SBTTag>` in `Types.h` |
| `ICommandBuffer::traceRaysWithSBT(SBTHandle, uint32_t w, h, d)` | Extended dispatch; Null backend no-op |
| `Renderer::setShaderBindingTable` / `shaderBindingTable()` | Non-owning SBT slot; used in RT dispatch when valid, falls back to `traceRays` if null |
| Null-backend tests | `test_RTShaderBindingTable.cpp` — 8 tests |

## Out of Scope for v0.6

- Full path tracer (multiple-bounce lighting, BRDF importance sampling).
- macOS MoltenVK RT path.
- Multi-GPU rendering.
- AMD FSR 4 frame generation.

---

## Exit Criteria

1. `NeuralRendererFactory::create(FSR3, device)` returns non-null on all platforms; reports
   `UpscalerBackend::FSR3` when FidelityFX SDK is present.
2. `DLSSPlugin::denoise()` and `upscale()` invoke the NGX evaluation API on Tier-1 hardware
   when feature handles are valid.
3. `IDevice::createShaderBindingTable` returns a valid handle on Null and Vulkan backends.
4. `traceRaysWithSBT` dispatch fires on Tier-1 hardware without crash.
5. All new Null-backend tests pass (target: +15 tests from Tracks 1 + 3).
6. Null-backend full suite remains at 100% (no regressions).
