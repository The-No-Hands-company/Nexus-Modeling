# v0.7 Planning — VRS, Volumetric Lighting, MSAA

Status: **Planning**. Three self-contained rendering tracks that improve visual quality
and perf-scalability on existing hardware without new SDK dependencies.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.6-planning.md](v0.6-planning.md).

---

## Motivation

v0.6 closed the RT pipeline loop with SBTs and wired the NGX evaluation path. The
render loop now has a clean structure but is missing three production-quality passes
used by every shipping title:

1. **Variable-Rate Shading (VRS)** — `caps().variableRateShading` is already plumbed
   but never invoked. Per-tile shading-rate control cuts GPU cost on low-detail regions
   while preserving quality at edges and focus points.
2. **Volumetric Lighting** — a compute-based froxel raymarch pass that adds atmospheric
   scattering (fog, god-rays) without path tracing. Completely self-contained; no new
   caps required.
3. **MSAA Resolve** — the GBuffer render targets are currently single-sample. A
   configurable MSAA sample count with a resolve pass before the composite brings
   hardware anti-aliasing to the rasterize mode.

## Scope

### Track 1 — Variable-Rate Shading

| Deliverable | Notes |
|---|---|
| `ShadingRate` enum | 1×1, 1×2, 2×1, 2×2, 4×2, 4×4 variants |
| `RendererSettings::enableVRS` | Gates VRS dispatch; default `false` (opt-in) |
| `RendererSettings::defaultShadingRate` | Coarse rate applied to non-focused regions |
| `FrameStats::vrsActive` | `true` when VRS was set this frame |
| `ICommandBuffer::setFragmentShadingRate` | Sets per-draw coarse rate; no-op on Null |
| `Renderer::render()` VRS block | Before geometry pass: when `enableVRS && caps().variableRateShading`, call `setFragmentShadingRate`; reset to 1×1 after geometry |
| Null-backend tests | `test_VariableRateShading.cpp` — 8 tests |

### Track 2 — Volumetric Lighting Pass

| Deliverable | Notes |
|---|---|
| `VolumetricSettings` struct | `enabled`, `fogDensity`, `scatteringCoeff`, `extinctionCoeff`, `froxelSlices`, `froxelResolutionDivisor` |
| `Renderer::setVolumetricSettings` / `volumetricSettings()` | Stored on renderer |
| `FrameStats::volumetricActive` | `true` when the froxel pass ran |
| `FrameStats::volumetricFroxelCount` | Total froxels dispatched |
| Compute dispatch in `render()` | After shadow pass, before composite: when `settings.enabled`, dispatch froxel integration shader via `cmd.dispatch()` |
| Null-backend tests | `test_VolumetricLighting.cpp` — 8 tests |

### Track 3 — MSAA Resolve

| Deliverable | Notes |
|---|---|
| `RendererSettings::msaaSamples` | 1 (off), 2, 4, 8 — clamped to `caps().maxMsaaSamples` |
| `DeviceCapabilities::maxMsaaSamples` | New cap field; Null backend reports 1 |
| `FrameStats::msaaSamples` | Sample count actually used this frame |
| `ICommandBuffer::resolveTexture` | Resolves multi-sample attachment to single-sample output |
| Renderer multi-sample GBuffer allocation | GBuffer textures allocated at `msaaSamples` count; resolve fired before composite |
| Null-backend tests | `test_MSAAResolve.cpp` — 7 tests |

---

## Out of Scope for v0.7

- Full path tracer (multi-bounce BRDF).
- AMD FSR 4 frame generation.
- macOS MoltenVK RT path.
- Screen-space global illumination (SSGI).

---

## Exit Criteria

1. `RendererSettings::enableVRS` gates `setFragmentShadingRate` correctly; `vrsActive`
   set to `true` when VRS fires.
2. `VolumetricSettings::enabled` triggers the froxel compute dispatch; `volumetricActive`
   and `volumetricFroxelCount` populated.
3. `RendererSettings::msaaSamples > 1` allocates multi-sample GBuffer; `resolveTexture`
   called before composite; `msaaSamples` stat reflects clamped count.
4. All new Null-backend tests pass (target: +23 tests from all three tracks).
5. Null-backend full suite remains at 100%.
