# v0.8 Planning — Screen-Space Post-Processing Passes

Status: **Planning**. Wires the three silent post-processing flags
(`enableAO`, `enableSSR`, `enableBloom`) that have been in `RendererSettings`
since v0.2 but had no compute dispatch or FrameStats coverage.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.7-planning.md](v0.7-planning.md).

---

## Motivation

`RendererSettings` has carried `enableAO = true`, `enableSSR = false`, and
`enableBloom = true` as dead flags since the initial renderer skeleton. Three
production-quality screen-space passes are entirely compute-based, require no
new device caps, and close the gap between "declared" and "dispatched":

1. **SSAO** — horizon-based ambient occlusion from the GBuffer depth+normals.
2. **SSR** — screen-space reflection rays marched in view space against the depth
   buffer; blended with the composite pass output.
3. **Bloom** — multi-pass Kawase downsample + bicubic upsample chain with
   configurable threshold and intensity.

---

## Scope

### Track 1 — Screen-Space Ambient Occlusion (SSAO)

| Deliverable | Notes |
|---|---|
| `AOSettings` struct | `radius`, `bias`, `sampleCount`, `blurPasses` |
| `Renderer::setAOSettings` / `aoSettings()` | Stored on renderer |
| `FrameStats::aoActive` | `true` when SSAO dispatch ran |
| `FrameStats::aoSampleCount` | Hemisphere samples used this frame |
| Compute dispatch in `render()` | After GBuffer pass; `cmd.dispatch` on depth+normals |
| Null-backend tests | `test_ScreenSpaceAO.cpp` — 8 tests |

### Track 2 — Screen-Space Reflections (SSR)

| Deliverable | Notes |
|---|---|
| `SSRSettings` struct | `maxRaySteps`, `stepSize`, `thickness`, `fadeDistance` |
| `Renderer::setSSRSettings` / `ssrSettings()` | Stored on renderer |
| `FrameStats::ssrActive` | `true` when SSR dispatch ran |
| `FrameStats::ssrRayCount` | Total rays dispatched (width × height when active) |
| Compute dispatch in `render()` | After GBuffer pass; before composite |
| Null-backend tests | `test_ScreenSpaceReflections.cpp` — 8 tests |

### Track 3 — Bloom Post-Process

| Deliverable | Notes |
|---|---|
| `BloomSettings` struct | `threshold`, `intensity`, `radius`, `passes` |
| `Renderer::setBloomSettings` / `bloomSettings()` | Stored on renderer |
| `FrameStats::bloomActive` | `true` when bloom dispatch chain ran |
| `FrameStats::bloomPassCount` | Downsample + upsample pass count this frame |
| Compute dispatch in `render()` | After composite pass; `cmd.dispatch` per mip level |
| Null-backend tests | `test_BloomPostProcess.cpp` — 8 tests |

---

## Out of Scope for v0.8

- SSGI (screen-space global illumination).
- Full path tracer.
- Depth-of-field / motion blur passes.
- AMD FSR 4.

---

## Exit Criteria

1. `enableAO = true` triggers SSAO dispatch; `aoActive` and `aoSampleCount` populated.
2. `enableSSR = true` triggers SSR dispatch; `ssrActive` and `ssrRayCount` populated.
3. `enableBloom = true` triggers bloom chain; `bloomActive` and `bloomPassCount` populated.
4. All three flags default to their existing values (AO=true, SSR=false, Bloom=true) —
   no behavior change for callers who never set them explicitly.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
