# v0.22 Planning — Neural Radiance Fields, Gaussian Splatting Spectral, Light-Field Display

Status: **Planning**. Three passes completing the neural and implicit-representation
rendering stack deferred from v0.21.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.21-planning.md](v0.21-planning.md).

---

## Motivation

v0.21 delivered plenoptic light-field rendering, coherent wave optics, and spectral
participating media. Three items were explicitly deferred:

1. **Neural Radiance Fields (NeRF) Rendering** — evaluates a compact MLP-based radiance
   field via ray marching on the GPU; enables photorealistic novel-view synthesis from
   a trained NeRF asset without explicit geometry.
2. **Gaussian Splatting Spectral Extensions** — extends the existing Gaussian splat pass
   with per-splat spectral colour coefficients, enabling spectrally-accurate splatting
   within multi-spectral rendering pipelines.
3. **Light-Field Display Output** — generates sub-view arrays for lenticular and
   holographic display targets; encodes the plenoptic capture into a per-view tile
   layout suitable for glasses-free 3D display panels.

## Exit Criteria

1. `enableNeRF` triggers ray march dispatch; `neRFActive` and `neRFMarchSteps` populated.
2. `enableGaussianSplatSpectral` triggers spectral expand; `gaussianSplatSpectralActive`
   and `gaussianSplatSpectralBandCount` populated.
3. `enableLightFieldDisplay` triggers view-tile encode; `lightFieldDisplayActive` and
   `lightFieldDisplayViewCount` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
