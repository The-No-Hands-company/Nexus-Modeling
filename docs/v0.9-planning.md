# v0.9 Planning — Screen-Space Post-Processing Passes (Continued)

Status: **Planning**. Completes the post-processing suite left out of v0.8 scope:
depth of field, motion blur, and tone mapping / HDR pipeline.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.8-planning.md](v0.8-planning.md).

---

## Motivation

v0.8 shipped SSAO, SSR, and Bloom. Three more production-quality screen-space passes
remain before the post-processing suite is complete:

1. **Depth of Field (DoF)** — circle-of-confusion from the depth buffer; separable
   bokeh blur in 8×8 tiles.
2. **Motion Blur** — per-pixel velocity-vector integration sampled from the GBuffer
   velocity attachment; dispatched after composite.
3. **Tone Mapping / HDR** — exposure + ACES/Reinhard LUT applied to the final
   HDR composite buffer before presentation.

---

## Scope

### Track 1 — Depth of Field (DoF)

| Deliverable | Notes |
|---|---|
| `DoFSettings` struct | `focalDistance`, `focalRange`, `maxCoC`, `sampleRadius` |
| `Renderer::setDoFSettings` / `dofSettings()` | Stored on renderer |
| `FrameStats::dofActive` | `true` when DoF dispatch ran |
| `FrameStats::dofSampleCount` | Total CoC samples used this frame |
| Compute dispatch in `render()` | After composite; gated on `enableDoF` |
| Null-backend tests | `test_DepthOfField.cpp` — 8 tests |

### Track 2 — Motion Blur

| Deliverable | Notes |
|---|---|
| `MotionBlurSettings` struct | `shutterAngle`, `sampleCount`, `maxBlurRadius` |
| `Renderer::setMotionBlurSettings` / `motionBlurSettings()` | Stored on renderer |
| `FrameStats::motionBlurActive` | `true` when motion blur dispatch ran |
| `FrameStats::motionBlurSampleCount` | Velocity samples integrated this frame |
| Compute dispatch in `render()` | After DoF pass; reads GBuffer velocity |
| Null-backend tests | `test_MotionBlur.cpp` — 8 tests |

### Track 3 — Tone Mapping / HDR

| Deliverable | Notes |
|---|---|
| `ToneMappingSettings` struct | `exposure`, `whitePoint`, `operator_` (`ACES`/`Reinhard`/`Linear`) |
| `ToneMappingOperator` enum | `Linear=0`, `Reinhard=1`, `ACES=2` |
| `Renderer::setToneMappingSettings` / `toneMappingSettings()` | Stored on renderer |
| `FrameStats::tonemapActive` | `true` when tonemap dispatch ran |
| `FrameStats::tonemapOperator` | Enum value used this frame |
| Compute dispatch in `render()` | Final pass before present; always active when HDR enabled |
| Null-backend tests | `test_ToneMapping.cpp` — 8 tests |

---

## Out of Scope for v0.9

- AMD FSR 4 upscaler integration.
- Subsurface scattering.
- Lens flares / chromatic aberration.

---

## Exit Criteria

1. `enableDoF = true` triggers DoF dispatch; `dofActive` and `dofSampleCount` populated.
2. `enableMotionBlur = true` triggers motion blur dispatch; `motionBlurActive` and
   `motionBlurSampleCount` populated.
3. `enableToneMapping = true` triggers tonemap pass; `tonemapActive` and
   `tonemapOperator` populated.
4. All three flags default to off (non-breaking for existing callers).
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
