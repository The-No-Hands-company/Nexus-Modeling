# v0.4 Planning — Live DLSS4/XeSS Integration on Vulkan RT Hardware

Status: **Planning**. Targets the first release that requires physical GPU hardware with RT support.

Reference: [ROADMAP.md](../ROADMAP.md), [month-22-checklist.md](month-22-checklist.md).

---

## Motivation

M4 delivered the full RT production-path API, tested and gated on Null backend. The remaining
gap is live integration: plugging DLSS 4 and XeSS into the `INeuralRenderer` interface on a
real Vulkan device with `rayTracingTier >= 1` and running the end-to-end frame pipeline.

v0.4 closes that gap by adding:
1. A Vulkan-backed `INeuralRenderer` implementation for DLSS 4 and XeSS.
2. Live CI coverage on GPU-capable runners (rayTracingTier ≥ 1).
3. A hardware perf gate asserting realistic upscaler overhead on real hardware.

---

## Scope

### Track 1 — VulkanNeuralRenderer (DLSS 4 / XeSS)

| Deliverable | Notes |
|---|---|
| `VulkanNeuralRenderer` class | Implements `INeuralRenderer`; backend selected at construction (DLSS4, XeSS, OIDN) |
| DLSS 4 integration | Wraps DLSS SDK `NVSDK_NGX_DLSS_EvaluateFeature`; exposes `UpscalerBackend::DLSS4` and `DenoiserBackend::DLSS4` |
| XeSS integration | Wraps `xessExecute`; exposes `UpscalerBackend::XeSS` and `DenoiserBackend::XeSS` |
| `NeuralRendererFactory::create(backend, device)` | Factory returning `std::unique_ptr<INeuralRenderer>` |
| CMake options | `NEXUS_ENABLE_DLSS` / `NEXUS_ENABLE_XESS` already exist; wired to new impl |
| `nexus_neural` link target | Already linked to `nexus_kernel_tests`; needs the real impl behind the flag |

### Track 2 — Live RT Dispatch Tests

| Deliverable | Notes |
|---|---|
| `test_VulkanRTDispatch.cpp` | GoogleTest file, guarded by `NEXUS_BACKEND_VULKAN` + RT caps skip |
| `RTDispatch.TraceRaysRunsOnTier1Device` | Verifies `FrameStats::rtReflectionsActive == true` on real hardware |
| `RTDispatch.DLSSUpscalerActiveOnVulkan` | End-to-end: denoiser + upscaler + RT in one frame |
| `RTDispatch.XeSSUpscalerActiveOnVulkan` | Same for XeSS backend |
| Skip contract | All tests emit `[  SKIPPED ]` when `rayTracingTier == 0` or relevant SDK not compiled in |

### Track 3 — Hardware Perf Gate

| Deliverable | Notes |
|---|---|
| `perf_smoke` hardware mode | `--backend vulkan` flag; reports per-frame GPU time from `FrameStats::gpuTimeMs` |
| `KernelPerfSmoke.VulkanRT` ctest entry | 120 frames, RT enabled, asserts avg GPU time < 33 ms (30 fps floor) |
| `KernelPerfSmoke.DLSSSteadyState` | 64 frames, DLSS enabled, asserts upscaler overhead < 4 ms avg |

---

## Prerequisites

| Requirement | Notes |
|---|---|
| Vulkan device with `VK_KHR_ray_tracing_pipeline` | CI runner with NVIDIA RTX or AMD RDNA 2+ |
| DLSS SDK (NVIDIA) | `NEXUS_ENABLE_DLSS=ON`; SDK path via `DLSS_SDK_DIR` CMake variable |
| XeSS SDK (Intel) | `NEXUS_ENABLE_XESS=ON`; SDK path via `XESS_SDK_DIR` CMake variable |
| OIDN CPU fallback | Already available; no new requirement |

---

## Out of Scope for v0.4

- Path tracer shader implementation (scene-level BVH, material evaluation, lighting).
- DLSS Ray Reconstruction (DLSS-RR) — deferred to v0.5.
- Multi-GPU / SLI configurations.
- macOS / MoltenVK RT support.

---

## Exit Criteria

1. `VulkanNeuralRenderer` compiles cleanly with `NEXUS_ENABLE_DLSS=ON` and `NEXUS_ENABLE_XESS=ON`.
2. All 3 live RT dispatch tests pass on a Tier-1 RT device.
3. `KernelPerfSmoke.VulkanRT` and `KernelPerfSmoke.DLSSSteadyState` pass.
4. Null-backend test suite still at 100% (no regressions).
5. M4 API contract unchanged — no public header removals or incompatible changes.
