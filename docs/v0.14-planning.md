# v0.14 Planning — Multi-Bounce RTGI, HBAO+, Variance Shadow Maps

Status: **Planning**. Three passes that harden indirect lighting quality and shadow
fidelity to production-grade standards for large, complex scenes.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.13-planning.md](v0.13-planning.md).

---

## Motivation

v0.13 delivered single-bounce RTGI, atmospheric scattering, and light shafts. Three
quality improvements remain before indirect lighting and shadows reach production fidelity:

1. **Multi-Bounce RTGI** — extends the v0.13 RTGI pass beyond single-bounce; each
   additional bounce re-uses the irradiance cache from the previous bounce, amortised
   over frames via temporal reprojection. Explicitly deferred from v0.13.
2. **Horizon-Based Ambient Occlusion (HBAO+)** — replaces the v0.8 hemisphere-sample
   SSAO with a sliced horizon-angle integration; produces bent normals and more accurate
   occlusion for architectural and organic geometry.
3. **Variance Shadow Maps (VSM)** — moment-based soft-shadow filter over the existing
   depth atlas; stores depth² alongside depth; Chebyshev upper-bound query replaces
   the hard PCF comparison.

## Scope

### Track 1 — Multi-Bounce RTGI Extension

| Deliverable | Notes |
|---|---|
| `RTGISettings::multiBounceFeedback` field | `float` in [0,1]; blends prev-frame irradiance into current bounce |
| `RTGISettings::temporalAlpha` field | Exponential blend weight for temporal reprojection accumulation |
| `FrameStats::rtgiBounceCount` | Bounces actually dispatched this frame (= `maxBounces`) |
| Extended dispatch in `render()` | Passes per-bounce irradiance ping-pong; gated on `maxBounces > 1` |
| Null-backend tests | `test_MultiBounceRTGI.cpp` — 8 tests |

### Track 2 — Horizon-Based Ambient Occlusion (HBAO+)

| Deliverable | Notes |
|---|---|
| `HBAOSettings` struct | `enabled`, `radius`, `angleBias`, `sliceCount`, `stepCount` |
| `Renderer::setHBAOSettings` / `hbaoSettings()` | Stored on renderer |
| `RendererSettings::enableHBAO` | Gate flag; independent of legacy `enableAO` |
| `FrameStats::hbaoActive` | `true` when HBAO+ dispatch ran |
| `FrameStats::hbaoSampleCount` | `width × height × sliceCount × stepCount` |
| Compute dispatch in `render()` | After GBuffer; 8×8 tiles |
| Null-backend tests | `test_HBAO.cpp` — 8 tests |

### Track 3 — Variance Shadow Maps (VSM)

| Deliverable | Notes |
|---|---|
| `VSMSettings` struct | `enabled`, `blurRadius`, `lightBleedReduction`, `minVariance` |
| `Renderer::setVSMSettings` / `vsmSettings()` | Stored on renderer |
| `RendererSettings::enableVSM` | Gate flag; augments existing shadow pipeline |
| `FrameStats::vsmActive` | `true` when VSM blur + resolve ran |
| `FrameStats::vsmCascadeCount` | Shadow cascades processed through VSM this frame |
| Compute dispatch in `render()` | After shadow pass; Gaussian blur of depth/depth² atlas |
| Null-backend tests | `test_VarianceShadowMaps.cpp` — 8 tests |

---

## Out of Scope for v0.14

- Cascaded VSM with per-cascade blend weights (deferred to v0.15).
- HBAO+ bent-normal export for specular occlusion.
- RTGI reservoir resampling (ReSTIR GI — deferred to v0.15).

---

## Exit Criteria

1. `RTGISettings::multiBounceFeedback` and `temporalAlpha` round-trip through accessor;
   `rtgiBounceCount` equals `maxBounces` when RTGI active.
2. `enableHBAO` triggers HBAO+ dispatch; `hbaoActive` and `hbaoSampleCount` populated.
3. `enableVSM` triggers VSM blur+resolve; `vsmActive` and `vsmCascadeCount` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
