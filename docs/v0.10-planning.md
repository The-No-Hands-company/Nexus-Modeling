# v0.10 Planning — Light Probes / IBL, OIT, AMD FSR 4

Status: **Planning**. Three independent tracks that were deferred from v0.8 and v0.9.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.9-planning.md](v0.9-planning.md).

---

## Motivation

v0.9 completed the screen-space post-processing suite. Three production deliverables
remain from the deferred backlog:

1. **Image-Based Lighting (IBL)** — prefiltered environment map sampled per-pixel from
   the GBuffer normal; contributes specular and diffuse ambient terms to the composite.
2. **Order-Independent Transparency (OIT)** — weighted blended OIT pass resolves
   transparent fragments without draw-order dependency.
3. **AMD FSR 4** — `NeuralBackend::FSR4` upscaler slot wired into `NeuralRendererFactory`;
   `FSR4Plugin` follows the same interface as `FSR3Plugin`.

---

## Scope

### Track 1 — Image-Based Lighting (IBL)

| Deliverable | Notes |
|---|---|
| `IBLSettings` struct | `enabled`, `intensity`, `diffuseScale`, `specularScale`, `mipLevels` |
| `Renderer::setIBLSettings` / `iblSettings()` | Stored on renderer |
| `FrameStats::iblActive` | `true` when IBL compute dispatch ran |
| `FrameStats::iblMipLevels` | Prefiltered mip levels sampled this frame |
| Compute dispatch in `render()` | After GBuffer transition; gated on `enableIBL` |
| Null-backend tests | `test_IBLighting.cpp` — 8 tests |

### Track 2 — Order-Independent Transparency (OIT)

| Deliverable | Notes |
|---|---|
| `OITSettings` struct | `enabled`, `maxLayers`, `weightScale` |
| `Renderer::setOITSettings` / `oitSettings()` | Stored on renderer |
| `FrameStats::oitActive` | `true` when OIT resolve pass ran |
| `FrameStats::oitFragmentCount` | Total transparent fragments accumulated |
| Compute dispatch in `render()` | After opaque composite; gated on `enableOIT` |
| Null-backend tests | `test_OITransparency.cpp` — 8 tests |

### Track 3 — AMD FSR 4 Upscaler

| Deliverable | Notes |
|---|---|
| `NeuralBackend::FSR4 = 7` | Extends stable enum; after FSR3=5, Bilinear=6 |
| `FSR4Plugin` | Header + implementation; same interface as FSR3Plugin |
| `NeuralRendererFactory::create(FSR4)` | Probes FFX entry points; falls back to FSR3 |
| `createNeuralRenderer` auto chain | FSR4 probed between FSR3 and OIDN_CPU |
| Null-backend tests | `test_FSR4Upscaler.cpp` — 7 tests |

## Out of Scope for v0.10

- Full real-time global illumination (ReSTIR GI).
- Path-guiding / online learning.
- Denoised IBL with SH projection.

---

## Exit Criteria

1. `enableIBL = true` triggers IBL dispatch; `iblActive` and `iblMipLevels` populated.
2. `enableOIT = true` triggers OIT resolve; `oitActive` and `oitFragmentCount` populated.
3. `NeuralBackend::FSR4` factory path returns non-null; SDK-absent Null backend falls back.
4. All three tracks: new flags default to off / stable values (non-breaking).
5. All new Null-backend tests pass (target: +23 tests).
6. Null-backend full suite at 100%.
