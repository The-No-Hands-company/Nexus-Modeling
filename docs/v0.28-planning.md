# v0.28 Planning — Delaunay Triangulation, Edge Collapse, Constraint Propagation

Status: **Planning**. Three geometry kernel production upgrades building on the
exact predicates and half-edge topology shipped in v0.26–v0.27.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.27-planning.md](v0.27-planning.md).

---

## Motivation

1. **Delaunay triangulation** is the foundational mesh quality primitive.
   The `inCircle` predicate shipped in v0.27 makes an exact Bowyer-Watson
   implementation straightforward. Without CDT, the remesh pipeline cannot
   guarantee triangle quality or insert constrained edges (e.g. sharp
   feature lines) without topology corruption.

2. **Edge collapse** is the only standard half-edge editing operation
   missing from v0.26's HalfEdgeMesh. Flip, split, and collapse form the
   minimal complete set for progressive mesh, LOD decimation, and mesh
   simplification pipelines (Garland-Heckbert QEM requires all three).

3. **Constraint propagation** — the v0.26 solver runs global iterative
   relaxation with no dependency ordering. For large graphs this wastes
   iterations re-solving already-converged sub-graphs. A topological-sort
   traversal (Kahn's algorithm on the constraint dependency DAG) propagates
   each constraint exactly once when the graph is acyclic, and reports
   specific cycle membership when it is not.

## Delaunay Invariant

After every insertion, every triangle in the triangulation satisfies the
Delaunay criterion: no other point lies strictly inside the circumcircle
of any triangle. Verified via `inCircle` after bulk triangulation in tests.

---

## Edge Collapse Link Condition

An edge (u, v) may be collapsed only if the intersection of the one-rings
of u and v contains exactly the two shared neighbours of the edge (i.e., the
two adjacent face apex vertices for interior edges, or one for boundary edges).
Violation of this condition would produce non-manifold topology.

## Exit Criteria

1. `Delaunay2D::triangulate` produces a triangulation satisfying the Delaunay
   criterion verified by `inCircle` for all triangle-point pairs.
2. `collapseEdge` preserves half-edge manifold invariants; rejected for
   link-condition violations.
3. `ConstraintGraph::buildDependencyOrder` produces a valid topological sort
   for acyclic graphs and populates `cycleMembers` for cyclic ones.
4. `ParametricSolver::solve` with `useTopologicalOrder=true` converges in
   one pass for DAG-structured constraint systems.
5. All new tests pass; full suite at 100%.
