# v0.17 Planning — Spectral Dispersion, Photon Mapping, Auto-Exposure

Status: **Planning**. Three passes completing the physically-based rendering stack
for spectral transport, global caustics, and perceptual luminance adaptation.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.16-planning.md](v0.16-planning.md).

---

## Motivation

v0.16 delivered ReSTIR PT, chromatic aberration, and screen-space caustics. Three
physically-based passes remain explicitly deferred or unaddressed:

1. **Hero Wavelength Spectral Dispersion** — extends ReSTIR PT with per-path hero
   wavelength sampling; enables accurate dispersion through glass, prisms, and thin
   films. Deferred from v0.16.
2. **Photon Mapping** — progressive photon map built via compute dispatch; enables
   global caustics and multi-bounce subsurface transport beyond screen-space range.
   Deferred from v0.16.
3. **Auto-Exposure / Eye Adaptation** — GPU luminance histogram over the rendered
   frame followed by an exponential EV adaptation curve; feeds `exposure` into the
   tone mapping pass each frame.

## Scope

### Track 1 — Hero Wavelength Spectral Dispersion

| Deliverable | Notes |
|---|---|
| `SpectralSettings` struct | `enabled`, `wavelengthSamples`, `dispersionScale`, `heroWavelengthNm` |
| `Renderer::setSpectralSettings` / `spectralSettings()` | Stored on renderer |
| `RendererSettings::enableSpectral` | Gate flag (default `false`) |
| `FrameStats::spectralActive` | `true` when spectral dispatch ran |
| `FrameStats::spectralWavelengthSamples` | Wavelength samples evaluated this frame |
| Compute dispatch in `render()` | After ReSTIR PT; one pass per wavelength sample |
| Null-backend tests | `test_SpectralDispersion.cpp` — 8 tests |

### Track 2 — Photon Mapping

| Deliverable | Notes |
|---|---|
| `PhotonMappingSettings` struct | `enabled`, `photonCount`, `maxBounces`, `gatherRadius` |
| `Renderer::setPhotonMappingSettings` / `photonMappingSettings()` | Stored on renderer |
| `RendererSettings::enablePhotonMapping` | Gate flag (default `false`) |
| `FrameStats::photonMappingActive` | `true` when photon trace + gather ran |
| `FrameStats::photonsEmitted` | Photons traced from all light sources this frame |
| Compute dispatch in `render()` | Photon emission pass + gather pass after GBuffer |
| Null-backend tests | `test_PhotonMapping.cpp` — 8 tests |

### Track 3 — Auto-Exposure / Eye Adaptation

| Deliverable | Notes |
|---|---|
| `AutoExposureSettings` struct | `enabled`, `minEV`, `maxEV`, `adaptationSpeed`, `targetLuminance` |
| `Renderer::setAutoExposureSettings` / `autoExposureSettings()` | Stored on renderer |

| `FrameStats::autoExposureEV` | Computed exposure value applied this frame |
| Compute dispatch in `render()` | After composite; histogram reduce then EV update |
| Null-backend tests | `test_AutoExposure.cpp` — 8 tests |

## Out of Scope for v0.17

- Full multi-spectral rendering (N-wavelength paths — deferred to v0.18).
- Bidirectional photon mapping (BDPT — deferred to v0.18).
- Histogram-based auto white balance.

---

## Exit Criteria

1. `enableSpectral` triggers spectral dispatch; `spectralActive` and
   `spectralWavelengthSamples` populated.
2. `enablePhotonMapping` triggers photon emission + gather; `photonMappingActive`
   and `photonsEmitted` populated.
3. `enableAutoExposure` triggers histogram + EV adaptation; `autoExposureActive`
   and `autoExposureEV` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
