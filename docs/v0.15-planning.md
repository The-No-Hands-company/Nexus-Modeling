# v0.15 Planning — ReSTIR GI, Cascaded VSM Blending, Lens Flare

Status: **Planning**. Three passes that push indirect lighting quality and shadow
transitions to production fidelity, plus a physically-based lens simulation pass.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.14-planning.md](v0.14-planning.md).

---

## Motivation

v0.14 delivered multi-bounce RTGI, HBAO+, and VSM. Three quality improvements were
explicitly deferred or remain unaddressed:

1. **ReSTIR GI** — spatiotemporal reservoir resampling for indirect lighting;
   dramatically improves sample efficiency for scenes with many area lights and
   indirect paths. Deferred from v0.14.
2. **Cascaded VSM Blending** — per-cascade blend weights with smooth cross-fade at
   cascade boundaries; eliminates the hard seam visible in the v0.14 VSM implementation.
   Deferred from v0.14.
3. **Lens Flare & Anamorphic Streaks** — post-process lens simulation: radial flare
   ghosts and horizontal anamorphic streak from bright screen-space sources; composited
   additively before tone mapping.

## Scope

### Track 1 — ReSTIR GI

| Deliverable | Notes |
|---|---|
| `ReSTIRSettings` struct | `enabled`, `spatialReuse`, `temporalReuse`, `reservoirSize`, `clampThreshold` |
| `Renderer::setReSTIRSettings` / `reSTIRSettings()` | Stored on renderer |
| `RendererSettings::enableReSTIR` | Gate flag (default `false`) |
| `FrameStats::restirActive` | `true` when ReSTIR pass ran |
| `FrameStats::restirReservoirCount` | `width × height × reservoirSize` reservoirs updated |
| Compute dispatch in `render()` | After RTGI gather; spatial + temporal reuse passes |
| Null-backend tests | `test_ReSTIRGI.cpp` — 8 tests |

### Track 2 — Cascaded VSM Blending

| Deliverable | Notes |
|---|---|
| `VSMSettings::cascadeBlendRange` field | Fraction of cascade extent used for cross-fade [0,1] |
| `VSMSettings::perCascadeWeights[4]` field | Per-cascade contribution weight array |
| `FrameStats::vsmBlendedCascadeCount` | Cascades with active cross-fade blending this frame |
| Extended dispatch in `render()` | Blend pass after H+V blur; one group per cascade boundary |
| Null-backend tests | `test_CascadedVSMBlending.cpp` — 8 tests |

### Track 3 — Lens Flare & Anamorphic Streaks

| Deliverable | Notes |
|---|---|
| `LensFlareSettings` struct | `enabled`, `ghostCount`, `streakLength`, `threshold`, `intensity` |
| `Renderer::setLensFlareSettings` / `lensFlareSettings()` | Stored on renderer |
| `RendererSettings::enableLensFlare` | Gate flag (default `false`) |
| `FrameStats::lensFlareActive` | `true` when lens flare pass ran |
| `FrameStats::lensFlareGhostCount` | Ghost sprites composited this frame |
| Compute dispatch in `render()` | After composite; before tone mapping |
| Null-backend tests | `test_LensFlare.cpp` — 8 tests |

---

## Out of Scope for v0.15

- Full spectral lens simulation with aperture diffraction.
- ReSTIR PT (path tracing variant — deferred to v0.16).
- Adaptive per-cascade VSM resolution.

---

## Exit Criteria

1. `enableReSTIR` triggers ReSTIR pass; `restirActive` and `restirReservoirCount` populated.
2. `VSMSettings::cascadeBlendRange` and `perCascadeWeights` round-trip through accessor;
   `vsmBlendedCascadeCount` populated when VSM active.
3. `enableLensFlare` triggers lens flare + streak pass; `lensFlareActive` and
   `lensFlareGhostCount` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
