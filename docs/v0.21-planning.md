# v0.21 Planning — Plenoptic Rendering, Coherent Wave Optics, Spectral Participating Media

Status: **Planning**. Three passes completing the advanced wave-optics and volumetric
spectral transport stack deferred from v0.20.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.20-planning.md](v0.20-planning.md).

---

## Motivation

v0.20 delivered Mueller matrix BSDFs, time-resolved phosphorescence decay, and
hyperspectral IBL. Three items were explicitly deferred:

1. **Plenoptic / Light-Field Rendering** — captures and renders the full 4D plenoptic
   function (2D position × 2D direction) for refocusing, parallax, and synthetic
   aperture effects in post-process; extends the camera model beyond pinhole/thin-lens.
2. **Coherent Wave Optics** — models diffraction and interference phenomena (thin-film
   interference, diffraction gratings, edge diffraction) using Fourier optics on the GPU;
   complements the polarisation stack with physical wave behaviour.
3. **Spectral Participating Media** — extends volumetric froxel pass with wavelength-
   dependent extinction, scattering, and emission coefficients; enables accurate rendering
   of coloured smoke, spectral sunsets, and fluorescent fog.

## Exit Criteria

1. `enablePlenoptic` triggers light-field capture dispatch; `plenopticActive` and
   `plenopticRayCount` populated.
2. `enableWaveOptics` triggers Fourier optics passes; `waveOpticsActive` and
   `waveOpticsPassCount` populated.
3. `enableSpectralMedia` triggers spectral froxel dispatch; `spectralMediaActive` and
   `spectralMediaBandCount` populated.
4. All flags default to off — no behavior change for existing callers.
5. All new Null-backend tests pass (target: +24 tests).
6. Null-backend full suite at 100%.
