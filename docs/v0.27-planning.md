# v0.27 Planning — Predicate Extensions, Mesh Solid Topology, CSG Boolean Upgrade

Status: **Planning**. Three geometry kernel correctness upgrades building on the
half-edge mesh and robust predicate foundation shipped in v0.26.

Reference: [ROADMAP.md](../ROADMAP.md), [v0.26-planning.md](v0.26-planning.md).

---

## Motivation

v0.26 shipped the half-edge topology layer and exact orient2d/orient3d predicates.
Three correctness gaps remain before the geometry kernel can be considered
production-grade for solid modeling:

1. **Robust predicate coverage** is incomplete. `inCircle` (2D Delaunay) and
   `inSphere` (3D Delaunay) were explicitly deferred from v0.26. These are required
   for constrained Delaunay triangulation (CDT) and Bowyer-Watson Delaunay
   refinement, both of which underpin mesh quality improvement pipelines.

2. **Mesh solid topology** is not formally analysed. There is no API to compute
   Euler characteristic, genus, or connected component count — the standard
   descriptors a solid modeler must report before performing any topological
   surgery. These can now be derived from the half-edge structure.

3. **Boolean operations** still use approximate floating-point ray-casting for
   inside/outside classification. With `orient3d` available the point-in-mesh
   test can be upgraded to winding-number arithmetic, which is both more robust
   and correct for non-convex meshes.

| Deliverable | Notes |
|---|---|

| `test_BooleanOperation.cpp` | 6 additional tests exercising non-convex cases |

## Topology Formula Reference

```
χ = V − E + F         (Euler characteristic, half-edge mesh)
E = halfEdgeCount / 2  (for closed manifold)
genus = (2 − χ) / 2   (closed orientable surface)
```

For open meshes with b boundary loops:
```
χ = V − E + F
genus = (2 − χ − b) / 2   (if orientable)
```

## Exit Criteria

1. `inCircle` and `inSphere` return the correct sign for near-degenerate inputs
   that naive floating-point arithmetic gets wrong.
2. `MeshTopologyAnalyser::analyse` computes correct Euler characteristic and genus
   for sphere (genus 0) and torus-topology meshes (genus 1).
3. `BooleanOperation::compute` correctly classifies inside/outside for a non-convex
   input mesh when `useWindingNumber = true`.
4. All new tests pass; full suite at 100%.
